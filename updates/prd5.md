### Layout Pattern: Shared Settings Header
- New `src/app/(dashboard)/settings/layout.tsx` renders the shared settings header **only on the hub (`/settings` exactly)** — for sub-routes (`general`, `team`, `billing`, `limits`) the layout renders `{children}` with no header so the existing internal titles of those pages stay clean.
- Header content (hub only):
  - `<h1>` page title from `t('settings.header.title')` ("Settings" / "Configurações")
  - `<p>` subtitle from `t('settings.header.subtitle')` ("Manage your account, team and billing" / "Gerencie sua conta, equipe e faturamento")
- Implementation: client component using `usePathname()`; if `pathname === '/settings'`, render header + `{children}`, else render `{children}` only.
- The breadcrumb from `AppLayout` already indicates the active sub-route, so removing the layout title for sub-routes does not hurt navigation.

### Hub Page Pattern (`page.tsx`)
- Use `usePathname()` from `next/navigation` to compute `isActive` per item.
- Render an `ItemGroup` containing 4 `Item asChild` rows.
- Each row:
  ```tsx
  <Item asChild variant="outline" size="default" className={cn(isActive && "bg-accent text-accent-foreground border-accent")}>
    <Link href={href}>
      <ItemMedia variant="icon"><Icon /></ItemMedia>
      <ItemContent>
        <ItemTitle>{t(titleKey)}</ItemTitle>
        <ItemDescription>{t(descKey)}</ItemDescription>
      </ItemContent>
      <ItemActions><ChevronRightIcon className="size-4 text-muted-foreground" /></ItemActions>
    </Link>
  </Item>
  ```
- Active state visual: `bg-accent` + `text-accent-foreground` + accent border.
- Static config array at top of file with 4 entries: `{ href, icon, titleKey, descKey }`.
